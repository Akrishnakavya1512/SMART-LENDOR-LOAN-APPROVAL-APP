Technical Architecture

Layered design for real-time loan eligibility prediction.

This page maps the project to the reference architecture: user entry, Flask request handling, machine learning training, model storage, prediction serving, and IBM Cloud deployment.

User Layer

  • Loan applicant submits profile details.
  • Credit officer or financial analyst reviews predictions.
  • Browser receives the prediction result.

Frontend Layer

  • home.html shows dashboard and model readiness.
  • predict.html collects applicant inputs.
  • submit.html renders approval or rejection output.
  • Responsive UI supports desktop and mobile workflows.

Flask Application Layer

  • app.py defines routes and POST handling.
  • Input validation rejects missing or invalid values.
  • Request data is converted into the model feature schema.
  • Result templates display decision, risk level, and confidence.

Prediction Engine

  • Flask loads the saved .pkl model artifact.
  • User input is converted into a single feature vector.
  • The model predicts loan approved or loan rejected.
  • Prediction probability is shown as confidence.

Machine Learning Pipeline

  • Dataset collection from the loan applicant CSV.
  • Missing values handled with median and mode imputation.
  • Categorical variables encoded with one-hot encoding.
  • Numerical fields scaled for model compatibility.
  • Train-test split with stratified sampling.
  • Decision Tree, Random Forest, KNN, and XGBoost training.
  • Accuracy, confusion matrix, classification report, and cross-validation.
  • Best model selection and serialized model storage.

Deployment Layer

  • IBM Cloud-ready Procfile and manifest.yml.
  • gunicorn app:app web process for cloud hosting.
  • Browser-based access for bank users.
